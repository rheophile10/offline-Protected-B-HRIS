How to use the Offline HRIS

Short screen recordings of every task. Each clip is an automated Playwright test running against the real built app — so the guide can never drift from the product.

▶ Open the live demo ⭳ Download index.html
It's one file. Download index.html, double-click it, and everything below works offline — no server, no install, no admin rights. The demo passphrase can be anything you like; it encrypts whatever you export in that session.

1 · Start a session US-1

As a planner, open the app and load today's data behind a passphrase.

  1. Open the file — you land on a locked gate.
  2. Keep Demo data, type any passphrase, and choose Start session.
  3. Pick your operator identity. You arrive at the dashboard: 27 officers and the biggest staffing gaps.

2 · Add an officer US-2

Add a new hire; the establishment and dashboard update instantly.

  1. Go to Officers → + Add officer.
  2. Enter a badge number and name, then Save.
  3. The roster shows the new officer and the headcount rises to 28. The edit is attributed to you and audited.

3 · Assign an officer to a position US-3

Keep staffing levels and vacancy deficits current.

  1. Go to Assignments → + Assign officer.
  2. Pick an officer and a position, set the start date, and Assign.
  3. Staffing views recompute automatically.

4 · Query data & export an encrypted CSV US-4

Run ad-hoc SQL and share a figure — without plaintext ever touching disk.

  1. Open SQL Console, write a query, and press Run (or Ctrl/Cmd+Enter).
  2. Export the result — the only option is an encrypted .csv.enc, never a plain CSV.
  3. Recover it later with the same passphrase from Data & Security → Decrypt a file.

5 · Export changes & merge — no Node US-5

The coordinator merges everyone's offline edits inside the same app.

  1. An operator works, then Export my changes → an encrypted .hrischanges delta.
  2. A coordinator opens today's truth, goes to Data & Security → Merge operator changes, and selects the change files. cr-sqlite (a CRDT) converges everyone's edits.
  3. Export merged truth for tomorrow. No server, no Node, no admin rights.

7 · Recruit & hire an applicant US-7

Run a hiring pipeline and turn a successful applicant into an officer.

  1. Go to Recruitment. Open a competition on a vacant position and + Add applicant.
  2. Move applicants across the kanban: Applied → Screening → Interview → Background → Offer → Hired.
  3. On an Offer/Hired card, choose Hire →: it creates the officer record and an active assignment, and the headcount updates. Recruitment data syncs and merges like everything else.

8 · Training & compliance US-8

See what's expiring and keep officers qualified.

  1. Open Compliance. The cards show Expired / Expiring-in-90-days / Valid counts and how many officers are firearms-current.
  2. Click a card to filter the list (e.g. only expired), or filter by certification type.
  3. Hit Renew on a row (or + Record certification) — the expiry auto-fills from the recert interval, and the counts update.

9 · Leave & absence US-9

Track requests, approvals, and who's off.

  1. Open Leave — cards show who's on leave today, pending requests, and upcoming absences.
  2. + Request leave for an officer (the day count fills in automatically).
  3. Approve or Deny pending requests; the counts update.

10 · Workforce planning US-10

Project vacancies ahead of retirements.

  1. Open Planning. From each officer's length of service, it projects retirement-eligibility.
  2. The table shows the projected vacancy deficit at +1/+2/+3/+5-year horizons.
  3. See which officers are within five years of the pension milestone, so you can hire ahead.

11 · Audit trail US-11

See who changed what — attribution that survives the merge.

  1. Open Audit. The change log lists every edit with the operator, entity, and action.
  2. The session log records open / lock / import / export events.
  3. Filter by operator. Because the logs are CRRs, they travel with each changeset into the merge report.

12 · Officer file US-12

One place for an officer's contacts, performance, and conduct.

  1. Open Officer File and choose an officer.
  2. See their emergency contacts, performance reviews (with ratings), and conduct records in one view.
  3. Add a review, contact, or conduct record — or update a conduct disposition.

13 · Equipment register US-13

Track who holds each firearm, vehicle, radio, or camera.

  1. Open Equipment — cards show in-service / issued / available counts.
  2. Issue an available asset to an officer, or Return one that's out.
  3. Set an asset to Maintenance or Retired, or add new assets to the register.

6 · Lock the session US-6

Wipe decrypted data from memory when you step away.

  1. Click Lock session.
  2. The app returns to the gate and clears the in-memory database, key, and identity.
  3. Export your changes first — unsaved work is lost by design (no plaintext at rest).
